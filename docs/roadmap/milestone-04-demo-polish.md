## Objective

Turn the working system into a crisp, emotionally convincing hackathon story.

## Deliverables

- seeded end-to-end demo account
- scripted "coach meltdown" missed-run scenario
- polished mobile-first UI and dramatic pacing
- observability for the live demo path
- fallback-safe persona mode if risky branding or voice needs to be removed quickly
- final testing checklist for demo day

## Demo Story Beats

1. Show the goal and schedule setup.
2. Show the generated plan and today's mission.
3. Reveal the missed workout and growing debt.
4. Show escalating interventions across app, SMS, and call summary.
5. Show the agent restructuring the plan with a redemption workout.

## Team Workstreams

- design and frontend: visual polish, timing, and narrative clarity
- backend and worker: hardened seeded data path and stable demo toggles
- integrations: fallback behaviors for every external dependency
- ops: monitor logs, feature flags, and live environment readiness

## Completion Criteria

- demo can run start to finish from a clean environment
- external provider failures have graceful fallbacks
- every screen reinforces the personal AI agent story
- judges can understand the problem, solution, and social value quickly

## Risks

- spending too much time on edge-case hardening instead of the demo story
- introducing unstable live dependencies too late
