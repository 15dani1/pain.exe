# Team Roles And Ownership

## Purpose

Give the hackathon team a simple ownership model so multiple people can move at once.

## Suggested Lanes

- Product and pitch: refine story, social-good framing, and demo narrative
- Web frontend: landing page, onboarding, dashboard, and messaging surfaces
- Domain and backend: entities, plan generation contracts, and escalation policy logic
- Worker and integrations: queues, webhooks, calendar, telephony, voice, wearables
- Demo operations: seeded data, environment setup, analytics, testing, and fallback paths

## Coordination Rules

- lock domain contracts before deep UI branching
- keep provider payloads out of shared domain types
- document every new event name and state transition
- preserve the "personal AI agent" narrative in every demo-facing feature
